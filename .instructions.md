2. **MapScreen (Smart):** Fetches mock data, manages `viewMode` state (Map vs List), requests `expo-location` permissions on mount, and renders the user's location (Blue Dot) on a `react-native-maps` `<MapView>`. Safely filters and renders `<Marker>` elements.
3. **ViewToggle (Dumb):** Extracted UI component for switching Map/List views.
4. **RestaurantCard (Dumb):** Reusable, typed UI component for list items.
5. **Testing:** Fully passing TDD suites with extensive mocks for `supabase`, `react-native-maps`, and `expo-location`.
